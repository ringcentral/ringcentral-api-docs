Detailed Description

The RcvAudioDeviceManager class is a managing class to manage the audio devices and test them out.

◆ registerEventHandler:

- (void) registerEventHandler: (nullable id< RcvAudioDeviceEventHandler >)  handler

Registers the event handler.

The RingCentral video client SDK uses the RcvAudioDeviceEventHandler interface class to send audio device relevant event callbacks to the application, and the application inherits the methods of this interface class to retrieve those event callbacks.

handlerThe audio device event handler instance

◆ startPlaybackDeviceTest:

- (int64_t) startPlaybackDeviceTest: (nonnull NSString *)  audioFilePath

Starts the audio playback device test.

A successful call of startPlaybackDeviceTest triggers playing an audio file specified by the user. If the user can hear the voice that means the playback device works properly.

Note
Suggests calling this method before joining a meeting to make sure the audio works properly in the meeting.
Parameters
audioFilePathThe path of the audio file for the audio playback device test
  • Supported file formats: wav, mp3, m4a, and aac
  • Supported file sample rates: 8000, 16000, 32000, 44100, and 48000 Hz
Returns
0 means success or fails otherwise
