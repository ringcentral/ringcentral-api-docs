# Batch messaging message store

Each user extension can access its own message store to retrieve SMS and MMS message records, as well as batch message records associated with its direct phone number(s).

The message store provides access to both individual message details and batch messaging history, allowing developers to track message status, review delivery results, and manage message-related data for each user extension.

Use the APIs in the table below to access your batch messaging message store.

| API Endpoint | Description |
|-|-|
| [`/restapi/v2/accounts/~/extensions/~/sms/messages`](#read-batch-message-store) | Read the entire message store. |
| [`/restapi/v2/accounts/~/extensions/~/sms/batches`](#list-batches) | List all batches with essential info from the entire message store. |
| [`/restapi/v2/accounts/~/extensions/~/sms/batches/{batchId}`](#read-individual-batch) | Read a single batch basic info identified by the batch Id. |
| [`/restapi/v2/accounts/~/extensions/~/sms/status-summary`](#read-message-statuses) | Read aggregated message statuses from the message store. |
| [`/restapi/v2/accounts/~/extensions/~/sms/messages/{messageId}`](#read-individual-message) | Read a single message identified by the message Id. |

## Read batch message store

You can read the entire batch messaging message store then iterate through the `records` array in the response to parse each message's metadata. Use the `nextPageToken` to read the next page if the total number of records is greater than the maximum number of items per page.

To avoid loading the entire message store, which could be extremely large, you can use the following query parameters to control and limit the number of messages in a response. To read messages per batch, you can use the [List Batches](#list-batches) API to retrieve a list of batch Ids, then use a batch Id as a filter to read messages from that batch.

!!! important
    - Message records created by the enhanced business SMS and the high volume SMS API are replicated and stored in the batch messaging message store since November 2024. The message id of these replicated messages records are different from the original message store!
    - Outbound messages sent from a call queue's direct number using the batch messaging API are recorded in the batch messaging message store of the SMS handler extension of the call queue, who sent the messages.
    - Inbound messages sent to a call queue's direct number are recorded in the batch messaging message store under the call queue extension's message store. Any SMS handler assigned to a call queue SMS recipient can access the call queue extension's message store.


### Query parameters

| Filter | Description |
|-|-|
| `view` | Read the message store in "Simple" or "Detailed" mode. Detailed view will return the text message for each message item. The default value is "Simple". |
| `batchId`| Return only messages belong to a single batch identified by the batch id. |
| `timeFrom `| Return messages created after the start date and time, for example 2025-11-10T08:00:00.000Z. The default value is timeTo minus 24 hours. |
| `timeTo`| Return messages created before the end date time, for example 2025-11-11T08:00:00.000Z. The default value is current time. |
| `phoneNumber`| Return only messages which contain the specified phone numbers in either the "from" or the "to" field. Multiple phone numbers (up to 15) are accepted.|
| `direction` | Return messages with the specified direction. If not specified, both inbound and outbound messages are returned. Multiple values are accepted. |
| `perPage` | Indicate the page size (number of items). The default value is 1000. |
| `pageToken` | Return messages from a specified page. Note! Use a valid page token from the paging object. |

## List Batches

You can read batches and parse the response to get a batch id and other basic info such as the status, the number of recipients, the creation date and time etcetera. You can use the following query parameters to control and limit the number of batches in a response.

### Query parameters

| Filter | Description |
|-|-|
| `timeFrom` | Return batches created after the start date and time, for example 2026-11-10T08:00:00.000Z. The default value is timeTo minus 24 hours. |
| `timeTo` | Return batches created before the end date time, for example 2026-11-11T08:00:00.000Z. The default value is current time. |
| `from` | Return only batches sent from the specified phone number. |
| `status` | The status of a batch. Can be _Processing_ or _Completed_. |
| `perPage` | Indicate the page size (number of items). The default value is 1000. |
| `pageToken` | Return batches from a specified page. Note! Use a valid page token from the paging object. |

## Read Individual Batch

You can read a single batch with essential info such as the status, the number of recipients, the number of processed messages etcetera. This is helpful for reading and updating a batch progress in real-time if you send a large batch which would take several hours to be completed.

## Read Message Statuses

To get a quick report on message statuses in a batch or within a period of time, you can call this endpoint to get the aggregated message statuses and error codes (if any). If you need to detect each individual message status, use this [endpoint](#read-batch-message-store) instead.

### Query parameters

| Filter | Description |
|-|-|
| `batchId`| Return aggregated message statuses of messages belong to a single batch identified by the batch Id. If a batch Id is specified, the `timeFrom` and `timeTo` query params will be ignored if they are specified in the API request! |
| `timeFrom `| Return aggregated message statuses of messages created after the start date and time, for example 2026-11-10T08:00:00.000Z. The default value is timeTo minus 24 hours. |
| `timeTo`| Return aggregated message statues of messages created before the end date time, for example 2026-11-11T08:00:00.000Z. The default value is current time. |
| `phoneNumber`| Return aggregated message statuses of messages which contain the specified phone numbers in either the "from" or the "to" field. Multiple phone numbers are accepted. E.g. phoneNumber: ['16501112222','16502223333']|
| `direction` | Return aggregated message statuses of messages with the specified direction. If not specified, both inbound and outbound messages are returned. Multiple values are accepted.|

## Read Individual Message

You can read individual message details from the batch message store using a message Id. Message ids can be retrieved from the response of this [endpoint](#read-batch-message-store)

### Sample codes

=== "HTTP"
    ```http
    GET /restapi/v2/accounts/~/extensions/~/sms/messages/{messageId}
    ```

=== "JavaScript"
    ```javascript
    const RingCentral = require('@ringcentral/sdk').SDK

    // Instantiate the SDK and get the platform instance
    var rcsdk = new RC({
        clientId: "RC_APP_CLIENT_ID",
        clientSecret: "RC_APP_CLIENT_SECRET",
        server: "https://platform.ringcentral.com"
    });
  	var platform = rcsdk.platform();

    /* Authenticate a user using a personal JWT token */
    platform.login({ jwt: "RC_USER_JWT" })

    platform.on(platform.events.loginSuccess, async function(response){
      let messageId = "52511"
      read_individual_message(messageId)
    });

    platform.on(platform.events.loginError, function(e){
        console.log("Unable to authenticate to platform. Check credentials.", e.message)
        process.exit(1)
    });

    async function read_individual_message(messageId){
      try{
        let endpoint = `/restapi/v2/accounts/~/extensions/~/sms/messages/${messageId}`
        var resp = await platform.get(endpoint)
        var jsonObj = await resp.json()
        console.log(JSON.stringify(jsonObj))
      }catch(e){
        console.log(e.message)
      }
    }
    ```
